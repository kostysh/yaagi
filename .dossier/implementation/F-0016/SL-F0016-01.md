## Decisions / Assumptions Beyond The Current Model

- `POST /control/freeze-development` is now live and returns the governor result shape instead of the former `future_owned` placeholder.
- `decisionOrigin` is exposed as a route-level alias of durable `triggerKind` to match the compact API contract while preserving the database trigger terminology.
- `development_proposal_rate` now uses the delivered governor proposal count when the DB-backed source exists; fixture-based tests keep `null` to preserve F-0012 degraded-source behavior before the source is available.
- Proposal tables and constants were added in this slice only as canonical surfaces for later slices; proposal submission/decision behavior remains planned for `SL-F0016-02`.
