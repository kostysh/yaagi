## Intake findings

- В текущей системе Telegram уже работает как ingress через `F-0005`: long polling, config gating, allowed chat filtering, dedupe и fake Bot API smoke.
- Исходящий Telegram reply loop отсутствует: нет canonical `telegram.sendMessage` action, durable outbox/idempotency и egress evidence.
- Фича должна потреблять существующие owner seams, а не менять их на intake: ingress остаётся у `F-0005`, bounded actions у `F-0010`, security у `F-0018`, observability у `F-0023`, operator identity/RBAC у `F-0024`, support evidence у `F-0028`.

## Operator feedback

- Оператор уточнил, что Telegram бот должен быть доступен только ему как оператору, а не всем пользователям Telegram.
- Это зафиксировано как hard scope: V1 поддерживает только direct chat configured operator identity; публичные чаты, группы, каналы и произвольные allowlisted users не входят в scope.
