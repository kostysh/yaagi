# AGENTS.md

Этот файл задаёт правила для всех документов внутри каталога `docs/`.

## Язык документов

1. Канонический язык документации в `docs/` — русский.
2. Новые документы и правки в существующих документах должны писаться на русском языке.
3. Допустимо оставлять на английском только:
   - идентификаторы, коды статусов и protocol tokens (`F-0001`, `CF-001`, `candidate`);
   - имена API, CLI-команд, путей, файлов и переменных окружения;
   - кодовые блоки, типы, названия моделей, библиотек и внешних технологий;
   - узкие технические термины, если их перевод ухудшает точность.
4. Если в документе используется английский термин, по возможности рядом должна быть русская формулировка или контекст, объясняющий его смысл.
