# Support process gap

## Incident class

`support_process_gap`

## Detection signals

- No runbook covers the incident class or owner seam.
- Existing runbook requires a forbidden shortcut for normal operation.
- Support closure cannot identify required canonical evidence.

## Triage reads

- Read current runbook docs under `docs/support/runbooks/`.
- Read the affected feature dossier and owner/consumer matrix.
- Read support incident evidence refs already collected.

## Allowed actions

- Open a support process gap incident.
- Link backlog, source-review or dossier follow-up refs when support truth must change.
- Document human-only residual risk while the procedure is incomplete.

## Forbidden shortcuts

- Do not create hidden process truth in chat history, shell logs or local notes.
- Do not bypass owner seams to compensate for missing support procedure.
- Do not mark a process gap closed without a runbook or backlog follow-up ref.

## Escalation owner

`F-0028`

## Evidence requirements

- Support incident ref for the gap.
- Affected runbook or dossier refs.
- Backlog/source-review follow-up ref when canonical truth must change.

## Closure criteria

- A runbook update, dossier update or backlog follow-up is linked and residual risk is explicit.
